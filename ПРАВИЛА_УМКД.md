# Правила сборки УМКД из РУПД

Инструкция, по которой собран `УМКД_ПМ01_Готовый.docx`. Следуя ей, можно
собрать УМКД по любому другому модулю: достаточно скинуть новый РУПД (PDF/DOCX).
Весь контент — **на английском языке**.

---

## 0. Что нужно на входе

1. **РУПД модуля** (`РУПД ПМ XX.pdf`) — единственный источник правды по: часам,
   темам занятий, типам занятий, критериям, литературе, специальности.
2. **Эталон** (`умкд Ару Пример.pdf` / `Пример Бекет А. УМКД ПМ10 ХТП .docx`) —
   образец структуры и глубины контента.
3. **Методические рекомендации ТИПО 2025** — правила оформления.

---

## 1. Данные, которые вынимаются из РУПД

Извлекать текст из PDF: `pypdf` (в кодировке UTF-8, писать в файл, а не в stdout).

Из РУПД взять:
- **Название модуля**, **специальность**, **квалификация**, **разработчики**,
  утверждающий (зам. директора), кафедра/ЦМК, методист.
- **Часы по каждому РО (LO)** — из таблицы «Распределение часов по семестрам»
  и строк «Всего часов на РО».
- **Темы занятий** — полный список «Тема X.Y …» по каждому РО.
- **Тип занятия** для каждой темы (последняя колонка).
- **Критерии оценки** (по группам тем) — для контекста.
- **Список литературы** (базовая / дополнительная / интернет-ресурсы) — со стр. в конце.

> ⚠️ В РУПД бывают опечатки (строка «Всего часов на РО» может не совпадать с числом
> тем). **Всегда сверять: число тем × 2 = число часов.** При расхождении — уточнить
> у заказчика, какое значение верное (в ПМ01 РО 1.4 намеренно взят 36 ч / 18 уроков).

---

## 2. Ключевое правило часов и уроков

- **1 урок = 2 академических часа.**
- **Число уроков в РО = часы РО ÷ 2.**
- **Число строк в таблице академического календаря РО = число уроков** (+1 строка заголовка).
- Сумма часов всех РО = общий объём модуля.

Пример ПМ01: 30/60/36/36/90/36/36 ч → 15/30/18/18/45/18/18 уроков = **162 урока, 324 ч**.

---

## 3. Классификация урока: лекция или практика

По колонке «Тип занятия» в РУПД:

| Тип занятия в РУПД | Вид урока (kind) | Заголовок и нумерация |
|---|---|---|
| Изучение нового материала | **lecture** | `Lecture LO.Y` |
| Комбинированный урок | **lecture** | `Lecture LO.Y` |
| Урок совершенствования ЗУН | practical | `Practical work LO.Y` |
| Урок применения и закрепления знаний | practical | `Practical work LO.Y` |
| Урок комплексного применения знаний | practical | `Practical work LO.Y` |
| Урок комплексной оценки / закрепления и оценки ЗУН | practical | `Practical work LO.Y` |
| Урок обобщения и оценки знаний | practical | `Practical work LO.Y` |

- Если часы темы стоят в колонке **производственное обучение** → kind = **industrial**
  (заголовок раздела `INDUSTRIAL TRAINING MATERIALS`, урок `Industrial training task LO.Y`).
- Если чисто лабораторные → kind = **lab** (`Laboratory work LO.Y`).
- `Y` — номер темы из РУПД (Тема 1.5 → `1.1.5`; Тема 2.3 → `1.2.3`).

## 4. Структура документа (порядок разделов — по эталону)

```
Титульный лист
Лист согласования (APPROVAL SHEET)
Содержание (CONTENT) — с номерами страниц
Введение (INTRODUCTION)
I.  ACADEMIC CALENDAR OF MODULE     ← таблица весов LO
II. LO 1.1 <название РО>
    1. ACADEMIC CALENDAR            ← таблица всех уроков РО
    2. GLOSSARY
    3. LECTURE MATERIALS            ← только если в РО есть лекции
    4. PRACTICAL CLASS MATERIALS    ← (или LABORATORY / INDUSTRIAL TRAINING)
    5. LIST OF SOFTWARE AND MULTIMEDIA SUPPORT FOR TRAINING SESSIONS
    6. LIST OF LITERATURE
III. LO 1.2 …
…
VIII. LO 1.7 …
IX. Self-Study Topics for the Module
X.  Exam Questions
```

- Разделы модуля нумеруются **римскими** (I, II, … X).
- Подпункты внутри РО — **арабскими** (1–6).
- Каждый РО начинается с **разрыва страницы**.
- Раздел «LECTURE MATERIALS» пропускается, если в РО нет лекций (напр. РО, целиком
  из производственного обучения). Тогда после `2. GLOSSARY` идёт сразу `4.`.

---

## 5. Правила оформления (Методичка ТИПО 2025)

- **Шрифт:** Times New Roman.
- **Кегль:** тело 12 pt; заголовки разделов/титул 14 pt; код 11 pt (Courier New).
- **Интервал:** одинарный.
- **Выравнивание:** тело — по ширине (justify); заголовки — по центру.
- **Поля:** левое 25 мм, верхнее 20 мм, правое 15 мм, нижнее 20 мм.
  (в twips: 1 мм ≈ 56.7; left=1418, right=850, top=bottom=1134).
- **Нумерация страниц:** арабские цифры, **по центру снизу (footer), кегль 10**,
  поле `PAGE`. Титул входит в нумерацию.
- **Таблицы:** тонкие рамки со всех сторон (single, sz 4), шапка — жирная.
- **Красная строка** абзацев тела — 18 pt (first-line indent).

## 6. Академический календарь модуля (таблица весов LO)

3 колонки: `LO code | описание РО | Weight, %`.
- Первая строка — код и название модуля (жирным).
- По строке на каждый РО.
- Последняя строка — `Intermediate certification (exam)` = **40 %**.
- **Веса РО** распределяются пропорционально часам так, чтобы сумма весов РО = 60 %
  (остальные 40 % — экзамен). Округлять до целых, подгоняя сумму ровно к 60.
  Пример ПМ01: 6/11/7/7/15/7/7 = 60 %.

## 7. Таблица «ACADEMIC CALENDAR» внутри РО

4 колонки: `№ | Name of topics | Date of lesson | Type of activity`.
- Строк = число уроков РО. № — по порядку 1…N.
- Name of topics — тема из РУПД (перевод на английский).
- Date — пусто (заполняется преподавателем).
- Type of activity — тип занятия из РУПД (перевод):
  Study of new material / Combined lesson / Skills-improvement lesson /
  Application and consolidation of knowledge / Comprehensive assessment of knowledge и т.п.
- Над таблицей: `LO id + название`, `Instructor`, специальность, `Total hours: X (N lessons of 2 academic hours each)`.

---

## 8. Глубина контента каждого урока (как в эталоне)

**Лекция** (`Lecture LO.Y <тема>`):
- `Content` — план из 4–5 подпунктов.
- 12–18 содержательных абзацев реального текста (не «вода»), последний абзац —
  обобщающий («Thus, …»).
- `Control Questions` — ровно **5 вопросов**.

**Практика/лаба/производственное** (`Practical work / Laboratory work / Industrial training task LO.Y <тема>`):
- `Objective:` — цель работы.
- `Theoretical background` — 2–3 абзаца.
- `Equipment and materials` — список.
- `Procedure` — пронумерованные шаги (≈6).
- `Tasks for independent completion` — 3 задания.
- `Conclusion:` — вывод.
- `Control Questions` — 5 вопросов.

> Если урока в исходниках нет — **писать с нуля** строго по теме РУПД. Тема лекции/
> практики = точное название темы из РУПД.

**Глоссарий РО:** таблица `Term | Definition`, 8–13 терминов по теме РО, по алфавиту.

**Software / Literature:** единые списки (Software — из «средств обучения» РУПД;
Literature — из списка литературы РУПД, разбитого на core / additional / web).

**Self-Study Topics** (общий по модулю) и **Exam Questions** (общий, покрывает все РО) —
в конце.

---

## 9. Титул и лист согласования (из РУПД)

**Титул:** название колледжа; блок «APPROVED» (зам. директора + год); «Educational-
methodological complex of the module»; название модуля; специальность; квалификация;
`Atyrau, 2025`.

**Лист согласования:** протоколы ЦМК и Методсовета; председатель ЦМК; методист;
**все разработчики** из РУПД (через `___________ Ф.И.О.`).

## 10. Технический пайплайн (переиспользуемый)

Каталог `umkd_gen/`:

- `module.json` — данные уровня модуля (титул, введение, веса/экзамен, software,
  literature, self-study, exam questions).
- `lo_1_1.json … lo_1_7.json` — по одному файлу на РО. Схема:
  ```json
  {
    "id": "1.1", "title": "...", "weight": "6%", "hours": "30 hours",
    "glossary": [["Term","Definition"], ...],
    "lessons": [
      {"code":"1.1","title":"...","type_en":"Study of new material","kind":"lecture",
       "outline":[...], "body":["абзац", ...], "questions":["q1",... "q5"]},
      {"code":"1.5","title":"...","type_en":"...","kind":"practical",
       "objective":"...","theory":[...],"materials":[...],"procedure":[...],
       "tasks":[...],"conclusion":"...","questions":[...]}
    ]
  }
  ```
- `build_umkd.py` — рендерит `УМКД_ПМ0X_Готовый.docx` (сам проставляет позиции уроков,
  таблицы, разрывы, поля, номера страниц). Запуск: `python build_umkd.py`.

**Как собрать новый УМКД:**
1. Извлечь текст РУПД (`pypdf`), составить перечень тем/типов/часов.
2. Заполнить `module.json` под новый модуль.
3. Заполнить `lo_*.json` — по одному уроку на каждую тему РУПД, kind по разделу 3,
   контент по разделу 8.
4. `python build_umkd.py` → готовый docx.
5. Проверка (раздел 11).

## 11. Проверка перед сдачей

- Число строк каждого календаря = часы РО ÷ 2. ✔
- Сумма часов РО = объём модуля; веса РО + 40 % = 100 %. ✔
- Все темы РУПД присутствуют, коды уникальны (1.1…1.N). ✔
- У РО без лекций нет раздела «LECTURE MATERIALS». ✔
- Весь текст на английском; заголовки совпадают с эталоном. ✔
- Оформление: TNR, поля 25/20/15/20, номера страниц по центру снизу. ✔

## 12. Номера страниц для «Содержания»

python-docx страницы не считает — нужен реальный рендер (**Word COM** на Windows):
1. Открыть готовый docx **read-only**, `Repaginate()`, для каждого заголовка взять
   `Range.Information(3)` (wdActiveEndPageNumber), закрыть **без сохранения**.
2. Сопоставить страницы с пунктами Содержания по порядку и вписать.
   (скрипты: `umkd_gen/extract_pages.ps1`, `make_toc_pages.py`, `build_content_docx.py`).

> Правило: **исходный УМКД при извлечении страниц не изменять** — только чтение.
